# Hierarchy Breach Analysis

## Definition

- a hierarchy breach happens when the intended chain of command breaks and Master is pulled into execution management that should have been handled by Lead

## Common Causes

- Lead over-coding and under-checking
- vague dev scopes
- no current comms entry
- false repo-state assumptions
- slow response to blocked lanes

## Current Working Rule

- log the breach
- identify whether it was caused by silence, confusion, trust loss, or hallucinated state
- apply a corrective action in the same session, not only later
