# Session Friction Severity Map

## Low

- small wording drift
- delayed but recoverable note updates
- minor ambiguity that does not change work direction

## Mild

- lead forgets a dev check-in window
- blocker is logged late
- docs and code truth drift but no destructive action follows

## Critical

- dev bypasses Lead to reach Master
- Lead assigns work from false repo state
- hallucinated team or product truth changes execution
- token burn becomes uncontrolled because blocked work was not delegated or paused

## Rule

- low issues get correction
- mild issues get correction plus logged prevention
- critical issues get correction, logged consequence, and governance review
