# Session Smoothness Playbook

## Before Work Starts

- check current dev progress
- check current diff state
- check latest comms entry
- restate the standing roster and active lane ownership

## While Work Is Running

- update the user frequently with real state, not broad reassurance
- move blocked lanes to standby early
- keep the vault current so Master can see the truth without chasing it

## After A Phase Or Task Tranche

- log the result
- update the relevant control note
- run Lead self-reflection
- capture the training lesson for Orch

## Core Principle

- smooth sessions come from supervision, not from hoping the team will self-synchronize
