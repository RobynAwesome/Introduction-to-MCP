# Lead Self-Reflection Protocol

## When Lead Must Self-Reflect

- after each major phase boundary
- after each 20-task issuance
- after any hierarchy breach
- after any hallucination that affects planning or team state

## Required Questions

1. did I manage before I coded
2. did I check each active dev before starting my own tranche
3. did I keep the Master informed through notes instead of forcing direct intervention
4. did I burn tokens on work that should have been delegated
5. did I rely on verified truth or on remembered state
6. what is one correction I must make immediately

## Output Format

- one success
- one failure or risk
- one correction
- one training lesson for Orch
