# Robyn Operator Profile - Session 3

> Current-session operational profile for Robyn.
> This is an execution guide, not a psych evaluation.

## Current Session Read

Robyn is not asking for symbolic work.

She expects:

- actual consolidation, not cosmetic cleanup
- one document she can open and trust
- Obsidian-safe structure that respects how she works in the vault
- AI profiling that becomes reusable Orch training data
- directness, truth, and speed

## Current Friction Trigger

The fastest way to lose trust in this session is to do the technically narrow version of the task while ignoring the wider intent.

Example:

- request: organize the vault
- unacceptable response: move files into folders and stop
- acceptable response: map the information, fix stale paths, create indexes, consolidate status, and preserve Obsidian usability

## Working Preferences Observed In This Session

- she wants the system to infer the real goal, not hide behind literal compliance
- she values usable note architecture more than neat directories
- she wants profiling and memory built into the project, not treated as optional extras
- she expects AI agents to produce archival knowledge, not just code
- she prefers one strong source-of-truth note over many fragmented status notes

## Collaboration Guidance For Orch

- when Robyn says "organize", assume she means structure plus meaning
- when a vault is involved, index and cross-link before moving anything important
- when the work touches training, add the new lessons into the training corpus immediately
- when there is a mistake, correct it concretely and fold the correction into a reusable rule

## Best Interface For Robyn

- one "Now" note
- one dashboard
- one training index
- one open-issues note
- clear links from those notes to everything else
