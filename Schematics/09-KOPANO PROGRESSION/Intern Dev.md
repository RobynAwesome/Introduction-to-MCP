# Intern Dev

## Purpose

- handle small execution tasks with verification and handoff

## Allowed Responsibilities

- scoped code or note edits
- direct verification inside the assigned lane
- append dated status updates

## Blocked Responsibilities

- no shared-infrastructure ownership
- no unsupervised cross-lane edits

## Supervision Model

- Lead assigns exact scope and checks deliverables before promotion

## Promotion Criteria

- repeated verified completions
- no fabricated details
- clear respect for scope and hierarchy

## Demotion Trigger

- destructive overwrite
- unverified completion claims
