# Idea Entry Protocol

## Purpose

- stop ideas from entering the vault as raw excitement only
- make every idea prove why Orch should care

## Required Entry Sections

1. idea summary
2. why this idea is good for Orch
3. source session or source research
4. official source pack
5. competitor or platform comparison
6. `vs Orch`
7. three implementation options
8. lowest-risk option
9. disruptive-risk assessment
10. recommendation: `incubate`, `validate`, `defer`, or `reject`

## Extra Gate For Disruptive Ideas

- if the idea changes architecture, product identity, or the main demo path materially:
  - compare the idea against the current demo-safe route
  - define rollback cost
  - define migration cost
  - define the smallest possible proof slice first

## Rule

- if the idea cannot survive this process, it does not get promoted out of this folder
